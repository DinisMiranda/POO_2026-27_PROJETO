01 · Introdução

Contexto e Objetivos

Âmbito da Entrega

Entrega intermédia da UC Testes e Performance Web, integrada com Projeto II e Programação Web II.

Objetivo Principal

Definir e organizar o plano de testes com recurso ao Jira/Xray: Test Plan, ciclos Unit Tests e UAT.

Requisitos do Enunciado

  • Test Plan configurado no Jira/Xray
  • 2 ciclos de teste: Unit Tests e UAT
  • Requisitos funcionais configurados
  • Mínimo 5 requisitos não funcionais
  • Prioridade, componente e ferramenta por teste
  • Pré-condições, passos e resultados esperados

02 · Produto

Sistema de Controlo Ambiental

Aplicação web para monitorização e controlo remoto de parâmetros ambientais domésticos, assente numa arquitetura cliente-servidor com REST API.

Objetivos do Projeto

  • —REST API robusta com Node.js/Express
  • —Integração com sensores e atuadores IoT
  • —Painel de controlo web (frontend)
  • —Registo histórico de leituras e consumos
Monitorização
Temperatura e iluminação em tempo real
Controlo de Dispositivos
Controlo manual e automático de atuadores
Histórico de Dados
Consulta de leituras e ações anteriores
Gestão de Utilizadores
Painel de administração com permissões
03 · Arquitetura

Arquitetura do Sistema

Frontend

Interface web para utilizadores e administradores — consulta de dados e controlo de dispositivos.

Backend / REST API

Processa pedidos, aplica regras de negócio e devolve dados ao frontend.

Base de Dados

Armazena utilizadores, sensores, leituras, histórico, consumos e ações.

Sensores IoT

Recolha automática de dados ambientais: temperatura e iluminação.

Atuadores

Dispositivos controláveis: iluminação, ventilação e outros equipamentos.

Ferramentas de Teste

Jira, Xray, Postman, JMeter, Google Lighthouse e testes manuais.
